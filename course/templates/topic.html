{{ language.name }}: Array types and conversions between types

NumPy supports a much greater variety of numerical types than Python does. This section shows which are available, and how to modify an array's data-type.

NumPy numerical types are instances of numpy.dtype (data-type) objects, each having unique characteristics. Once you have imported NumPy using import numpy as np you can create arrays with a specified dtype using the scalar types in the numpy top-level API, e.g. numpy.bool, numpy.float32, etc.

These scalar types as arguments to the dtype keyword that many numpy functions or methods accept. For example:

>>> z = np.arange(3, dtype=np.uint8)
>>> z
array([0, 1, 2], dtype=uint8)

Array types can also be referred to by character codes, for example:

>>> np.array([1, 2, 3], dtype='f')
array([1., 2., 3.], dtype=float32)
>>> np.array([1, 2, 3], dtype='i')
array([1, 2, 3], dtype=int32)

Numerical Data Types

NumPy provides a wide range of numerical data types that can be used to create arrays. These include integers, floating-point numbers, complex numbers, and more.

Integer types in NumPy include various sizes from 8-bit to 64-bit integers, both signed and unsigned. Floating-point types include 16-bit, 32-bit, and 64-bit floats, with 64-bit being the standard double precision.

>>> # Integer types
>>> int8_array = np.array([1, 2, 3], dtype=np.int8)

Data Types for Strings and Bytes

NumPy also supports arrays of strings and bytes, with fixed-length or variable-length elements.

String arrays in NumPy can be created with specific character lengths, which can help optimize memory usage when working with textual data.

>>> # Fixed-length string array
>>> str_array = np.array(['apple', 'banana', 'cherry'], dtype='U10')
>>> str_array
array(['apple', 'banana', 'cherry'], dtype='

Best Practices

When working with NumPy data types, it's important to follow best practices to ensure efficient and correct code.

Choose the smallest data type that can adequately represent your data to minimize memory usage. Be mindful of potential overflow issues when working with integer types.
